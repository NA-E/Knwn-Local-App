Page Structure

Team app + Client dashboard (separate auth).

Team App (team_members auth)

  • Dashboard
    Role-aware landing. Admin: stats + stuck projects. Others: redirect to My Board.
    All roles
  • Clients
    • Client List
      Table: name, market, pod, status. Filters + search.
    • Client Detail
      Info, channels, contacts, team assignments, projects, links.
    • Add Client
      Form + triggers onboarding automation.
  • Projects
    • Full Pipeline
      Admin Kanban — all statuses, all projects.
      Admin, Strategist
    • My Board
      Role-specific Kanban.
      All roles
    • Project Detail
      Status actions, links (script/edit/thumbnail), versions, activity log.
  • Team
    • Team Members
      Table: name, role, pod, status.
    • Pods
      Cards: name, member count, client count.
  • Settings
    Manage users, create accounts, deactivate + off-board.
    Admin only

Client Dashboard (magic link auth)

  • /client/[token]
    • Upload
      Dropbox File Request link — always visible, one click.
    • Projects
      Read-only status list with friendly labels.
    • Reviews
      Dropbox review links when edit is sent to client.
    • Resources
      Brand docs, onboarding materials, FAQ.

Targeted Slack Notifications

4 critical-path notifications using the existing Slack service layer. Not a full notification system.

editing → ready_for_internal_review
Slack DM → assigned Senior Editor
Editor submitted V1/V2/V3 for review. Senior Editor needs to review quickly.
review_script → fix_script
Slack DM → project's Writer
Senior Writer rejected script. Writer needs to fix and re-submit.
ready_for_int_review → internal_adj_needed
Slack DM → project's Editor
Senior Editor rejected edit. Editor needs to fix feedback.
internal_adj_needed → ready_for_int_review
Slack DM → assigned Senior Editor
Editor re-submitted after fixes. Senior Editor reviews again.
Role-by-Role Experience

Who Sees What & Who Does What

Ten roles. Each scoped by DB query — not by UI hiding alone. RLS policies enforce visibility; these cards describe the experience each role gets inside the app.

Leadership
Admin Full Access
Clayton · Andrew · Paulo
What I See
  • All pages, all clients, all projects across all pods
  • All board columns: Idea → Posted / Scheduled + Cancelled
  • User management page (create/edit/deactivate team members)
  • Dashboard: stats, stuck projects, pod breakdowns
What I Do
  • Full CRUD on all entities (clients, projects, team members)
  • Create and deactivate team member accounts
  • Move any project to any valid status transition
  • Assign any role to any client
  • Trigger and monitor client onboarding automation
  • Undo: posted_scheduled → ready_to_post (Admin only)
My Filter
-- No filter. RLS grants full read+write.
SELECT * FROM projects; -- all rows visible
Strategy
Strategist Full Pipeline
Multiple per pod
What I See
  • All projects for clients assigned to me
  • All 17 board columns: Idea → Posted/Scheduled
  • Client detail pages for my assigned clients
What I Do
  • Create new projects for assigned clients
  • Move any project through valid status transitions
  • Assign writers to projects at Brief → Scriptwriting
  • Edit client info, add notes
  • Pause projects (any → on_hold), cancel projects
My Filter
SELECT p.* FROM projects p
JOIN client_assignments ca ON ca.client_id = p.client_id
WHERE ca.team_member_id = auth.uid()
AND ca.assignment_role = 'strategist'
Jr Strategist Strategist + Manager
Uses strategist slot in client_assignments
What I See
  • Same scope as Strategist — all projects for assigned clients
  • All 17 board columns including handoff columns
What I Do
  • Everything a Strategist can do
  • Everything a Manager can do: send scripts/edits, mark posted
  • Submit scripts (Writer), approve scripts (Sr Writer), approve edits (Sr Editor)
My Filter
-- Same as Strategist:
...WHERE ca.assignment_role = 'strategist'
Critical Gotcha

Handoff permissions must check team_role = 'jr_strategist', NOT assignment_role = 'manager'. Jr Strategists have no manager assignment row — any RLS policy joining through client_assignments for manager transitions will silently block them.

Client Management
Manager Handoff Columns
Client-facing liaison
What I See
  • Script Ready to Send — approved, waiting to go out
  • Script Sent to Client — watching for client response
  • Edit Ready to Send — approved, waiting to go out
  • Edit Sent to Client — watching for client response
  • Ready to Post — approved, awaiting scheduling
What I Do
  • Script Ready → Script Sent to Client
  • Script Sent → Client Uploaded (client filmed & uploaded)
  • Edit Ready → Edit Sent to Client
  • Edit Sent → Ready to Post or Client Adjustments Needed
  • Ready to Post → Posted/Scheduled
My Filter
...WHERE ca.assignment_role = 'manager'
AND p.status IN ('script_ready_to_send',
  'script_sent_to_client', 'edit_ready_to_send',
  'edit_sent_to_client', 'ready_to_post')
Writing
Senior Writer Review Gate
Rafael · Vivian
What I See
  • Review Script column only
  • Only scripts from writers I supervise (via supervised_by chain)
What I Do
  • Review Script → Script Ready to Send (approve)
  • Review Script → Fix Script (reject — requires feedback note)
My Filter
SELECT p.* FROM projects p
WHERE p.writer_id IN (
  SELECT id FROM team_members
  WHERE supervised_by = auth.uid())
AND p.status = 'review_script'
Writer Assigned Only
writer_id = me
What I See
  • Scriptwriting — my active writing work
  • Fix Script — scripts rejected by Senior Writer
What I Do
  • Scriptwriting → Review Script (submit)
  • Fix Script → Review Script (resubmit after fixes)
My Filter
SELECT * FROM projects
WHERE writer_id = auth.uid()
AND status IN ('scriptwriting', 'fix_script')
Editing
Senior Editor Review Gate
client_assignments.senior_editor
What I See
  • Ready for Internal Review column only
  • Only for clients where I'm the assigned Senior Editor
What I Do
  • Ready for Internal Review → Edit Ready to Send (approve)
  • Ready for Internal Review → Internal Adjustments Needed (reject — requires feedback note)
My Filter
SELECT p.* FROM projects p
JOIN client_assignments ca ON ca.client_id = p.client_id
WHERE ca.team_member_id = auth.uid()
AND ca.assignment_role = 'senior_editor'
AND p.status = 'ready_for_internal_review'
Editor Assigned Only
editor_id = me
What I See
  • Client Uploaded — new footage queue
  • Editing — active work
  • Ready for Internal Review — awaiting Sr Editor
  • Internal Adjustments Needed — Sr Editor feedback
  • Client Adjustments Needed — client feedback
What I Do
  • Client Uploaded → Editing (begin work)
  • Editing → Ready for Internal Review (submit + edit_url + edit_version)
  • Internal Adjustments → Ready for Internal Review (resubmit)
  • Client Adjustments → Editing (revision)
My Filter
SELECT * FROM projects
WHERE editor_id = auth.uid()
AND status IN ('client_uploaded', 'editing',
  'ready_for_internal_review',
  'internal_adjustments_needed',
  'client_adjustments_needed')
