- **Problem:** Building the desired visual dashboard in Notion proved overly complex.
- **Specific Blockers:** Notion cannot easily create the necessary relational properties for metrics like "videos launched vs. planned" per pod.
- **Solution:** Pivot to a table-based dashboard.
- **Rationale:** This is a faster, more reliable solution for the short term, as the company plans to migrate to a custom app. The focus is on accurate data tracking, not UI polish.

#### Data Integrity Issues
- The current dashboard data is unreliable, making it unusable for decision-making.
- **Calls Logged:** Most strategists show 0% calls logged. This is likely a tracking failure, not a lack of activity, as one strategist (Joe) shows >100% completion.

#### Key Metrics for Focus
- Calls Logged
- Videos Launched %
- Client Retention
- Pod Capacity
- **Rationale:** Ranking and CTR are less critical for this high-level view, as they tend to average out across pods. They can be tracked separately for individual strategist feedback.

### Client Onboarding & Pod Assignment

- **Onboarding Ownership:** The process is inconsistent, with no single owner.
  - **Current State:** Tasks (Notion page, Dropbox, welcome message) are handled by whoever is available (e.g., YouTube manager, Paulo).
  - **Goal:** Nourin is mapping the current process to identify gaps and opportunities for automation.
- **Pod Assignment Process:** This is a manual, case-by-case decision made by Clayton.
  - **Rationale:** The decision is based on subjective factors beyond raw capacity, such as a pod's current workload or recent onboarding volume. This prevents simply assigning to the pod with the lowest client count.

### Storage System Consolidation

- **Problem:** The current dual-system approach (Dropbox for footage, Google Drive for scripts) creates tool sprawl and disorganization.
- **Goal:** Consolidate into a single, structured system to improve organization and reduce access complexity.
- **Dropbox's Advantage:** Dropbox is preferred for its superior video review features (replay, comments) and robust automation capabilities.
- **Current Dropbox Structure:**
  - Clients upload footage via unique file request links.
  - An automation logs the upload date and uploader, improving tracking.
  - Folders are organized by client and contain sub-folders for A-roll, B-roll, edited videos, and assets.
