What we heard

Three problems that keep coming up

These aren't assumptions. Clayton and Paulo described these pain points in their own words.

PROBLEM 01

Nobody owns onboarding

Steps get missed. No checklist. No detection when something falls through. Whoever is available just does it.

PROBLEM 02

Client info is everywhere

Dropbox on one email. Google Drive on another. Slack for links. Four tools, four logins, no single view.

PROBLEM 03

Clients are in the dark

Upload links buried in Slack canvases. Review links sent ad-hoc. No project status. No self-service.

"There's not like a set person. It's whoever is available. We don't have it down to a super process thing at the moment."

Clayton & Paulo, on onboarding ownership

"One's in this drive that's in another email. One's in this Dropbox that's in another email and different people access it different ways. It's just a little annoying."

Clayton, on tool fragmentation

Recommendation 03

Replace Google Drive with In-App Doc Links

The app stores clickable links to Google Docs. The actual documents stay in Google Docs for editing. GDrive as a folder navigation layer becomes unnecessary.

"We basically use Google Drive because it has a great integration for docs. We don't use it for anything else."

Paulo, on why GDrive still exists

Before (Google Drive)

  • Navigate to GDrive (different email account)
  • Find the client folder
  • Open Scripts or Resources subfolder
  • Find the right document
  • GDrive "a bit messy" (Paulo)
  • Downloads fail for large folders (2GB zip limit)

After (App)

  • Open client profile in the app
  • Click the doc link
  • Google Docs opens in new tab
  • Same editing experience
  • One fewer tool to manage
  • Doc links also visible on client dashboard

Phased approach — no big-bang migration

Phase 1: Add doc link fields to client profiles and project records. Team pastes Google Doc URLs into the app. GDrive still exists but team navigates from the app instead.

Phase 2: Once the writing team confirms they're using app links (not GDrive navigation), stop creating new GDrive folders during onboarding. Existing folders stay as-is.

Build order & reasoning

Module 0+1: Auth & Clients Done

Foundation. Can't build anything without user accounts and client records.

Onboarding Automation Spec'd

Highest-impact problem (score: 0.86). Solves "nobody owns onboarding" immediately. Creates Slack + Dropbox + GDrive on client creation. Needs API credentials to go live.

Module 2: Production Pipeline (Kanban) Next

The core of daily operations. Project cards, status transitions, version tracking. Replaces the Notion task boards and edit submission workflow. Needed before client dashboard (which shows project status).

GDrive Replacement: Doc Links New

Low engineering effort (it's just URL fields on client profiles + project records). Can be built alongside Module 2. Eliminates one entire tool from the stack.

Client Dashboard New

Depends on Module 2 for project status data. Read-only portal: upload link, review links, project status, brand docs. Small scope with high client-facing impact.

Module 3+4: Role Boards & Admin Later

Per-role filtered views of the Kanban. Admin dashboard with team management. Builds on Module 2's foundation.
