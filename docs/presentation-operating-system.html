Section 7 — Build Order

Why this order

Each piece unlocks the next. Nothing is built out of sequence.

1

Auth + Client Management DONE

Login, roles, client records, team assignments. 80 real clients migrated from Notion. The foundation everything else builds on.

Foundation for everything else. Already live.
2

Onboarding Automation DESIGNED

Create a client → Slack channel + Dropbox folder + File Request link created automatically. Live progress modal. No missed steps.

Every client hits this. Captures onboarding timestamps from day one.
3

Production Pipeline (Kanban) + Doc Links

Project cards with status transitions, version tracking (V1/V2/V3), edit submission, review workflow. "Create Script" button auto-generates Google Docs. Resource links on client profiles. GDrive folders gone.

Replaces Notion boards and GDrive navigation. Every status change captures a timestamp. Writers click "Create Script" and a Google Doc appears. KPI data starts generating itself here.
4

Client Dashboard

Read-only portal per client: upload link (always visible), videos in review, project status, brand documents. Clients stop asking "where's my link?"

Depends on the Kanban for project status data. Captures client interaction timestamps (when they view, when footage arrives).
5

Role Boards + Admin + KPI Dashboards

Per-role filtered Kanban views. Admin tools. KPI dashboards that read from all the timestamps already being captured.

By this point, every action is in the system. KPI dashboards are queries against data that already exists.
